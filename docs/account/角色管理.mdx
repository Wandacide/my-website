# 角色管理

本文介绍管理员如何配置系统角色，并通过菜单权限、数据权限和用户分配，实现基于角色的访问控制。

## 功能简介

角色管理用于定义不同岗位在系统中的访问范围。管理员可通过「角色定义 - 菜单授权 - 数据授权 - 用户分配」的流程，为不同用户配置合适的功能权限和数据权限。

系统支持基于角色的访问控制（RBAC），可满足以下管理需求：

- 按岗位分配功能菜单；
- 按组织架构控制数据范围；
- 实现总部、部门、个人之间的数据隔离；
- 满足企业分级管理和权限合规要求。

## 操作前提

配置角色前，请确认：

1. 当前账号拥有「账号管理 - 角色管理」的新增、编辑、删除、分配用户等权限；
2. 已明确各岗位职责范围和所需系统权限；
3. 如需配置数据权限，需提前在「部门管理」中维护组织架构；
4. 如需分配用户，需提前在「创建账号」中维护用户账号。

## 操作流程

### 1. 进入角色管理

在左侧导航栏中，选择「账号管理」-「角色管理」，进入角色列表页面。

角色列表通常展示以下信息：

- 角色编号；
- 角色名称；
- 权限字符；
- 显示顺序；
- 状态；
- 创建时间；
- 操作入口。

[角色列表占位图]

> 图片占位：角色列表页面截图，建议展示角色编号、角色名称、状态和操作栏。

### 2. 新增角色

点击「新增」，打开「新增角色」弹窗。

需要填写的基础信息包括：

| 字段 | 是否必填 | 填写说明 |
| --- | --- | --- |
| 角色名称 | 是 | 输入角色显示名称，建议体现岗位职责，例如「培训管理员」。 |
| 权限字符 | 否 | 输入权限标识字符，用于系统代码级权限控制，例如 `admin`、`training_mgr`。 |
| 角色顺序 | 是 | 设置角色显示排序，数字越小排序越靠前。 |
| 状态 | 是 | 选择「启用」或「停用」。停用后该角色下用户将无法使用对应权限。 |
| 是否允许操作 | 是 | 控制该角色是否允许进行业务操作。 |
| 备注 | 否 | 填写角色用途、适用范围或管理说明。 |

[新增角色占位图]

> 图片占位：新增角色弹窗截图，建议展示基础字段和菜单权限区域。

### 3. 配置菜单权限

在「菜单权限」区域，勾选该角色可访问的功能模块。菜单权限决定用户登录后能看到哪些功能页面。

常见配置操作包括：

- 展开或收起全部菜单树；
- 全选或全不选；
- 开启父子联动，勾选父级时自动勾选子级；
- 按模块勾选审核管理、培训管理、考试管理、成绩管理、资源管理、统计分析、配置管理、账号管理等功能。

[菜单权限占位图]

> 图片占位：菜单权限树截图，建议展示父子级菜单和勾选状态。

:::tip 配置建议
建议遵循「最小权限原则」：只为角色勾选完成岗位职责所必需的功能模块，避免权限过大。
:::

### 4. 保存角色

确认角色基础信息和菜单权限无误后，点击「保存」。系统完成角色创建。

## 配置数据权限

数据权限用于控制角色可查看和操作的数据范围。创建角色后，可在角色列表中点击目标角色的「数据权限」入口进行配置。

系统通常提供以下数据权限范围：

| 数据权限 | 说明 | 适用场景 |
| --- | --- | --- |
| 全部数据权限 | 可查看和操作全量数据，不受组织架构限制。 | 超级管理员、总部管理员。 |
| 自定数据权限 | 自定义选择可查看的部门范围。 | 跨部门管理或专项负责人。 |
| 本部门数据权限 | 仅可查看本部门数据。 | 部门普通员工。 |
| 本部门及以下数据权限 | 可查看本部门及所有下级部门数据。 | 部门负责人、区域负责人。 |
| 仅本人数据权限 | 仅可查看本人创建或关联的数据。 | 数据敏感岗位、普通员工。 |

[数据权限占位图]

> 图片占位：数据权限配置弹窗截图，建议展示权限范围选项和部门树。

选择合适的数据范围后，点击「确定」保存。

## 分配用户

角色创建并配置权限后，需要将角色分配给具体用户。

操作方式：

1. 在角色列表中找到目标角色；
2. 点击操作栏的「分配用户」；
3. 进入已分配用户列表；
4. 点击「新增」，打开用户选择弹窗；
5. 勾选需要分配该角色的用户；
6. 点击「保存」。

## 管理已有角色

### 启用或停用角色

角色状态为「启用」时，角色权限正常生效；状态为「停用」时，该角色下用户将无法使用对应权限。

### 编辑角色

点击「编辑」可修改角色名称、权限字符、菜单权限等配置。修改后即时生效，已分配用户的权限会自动更新。

### 导出角色

点击「导出」可下载角色信息清单，用于权限审计、备份或定期盘点。

### 删除角色

点击「删除」可移除角色。

:::caution 删除前确认
如果角色下仍存在已分配用户，需先取消用户授权或将用户转移至其他角色，再删除该角色。
:::

## 关键字段说明

| 字段 | 说明 |
| --- | --- |
| 角色编号 | 系统自动生成的唯一角色标识，用于内部识别。 |
| 角色名称 | 角色显示名称，建议体现岗位职责。 |
| 权限字符 | 系统代码级权限标识，需保证唯一性。 |
| 显示顺序 | 角色在列表中的排序序号，数字越小排序越靠前。 |
| 状态 | 控制角色是否可用。启用表示生效，停用表示失效。 |
| 菜单权限 | 角色可访问的功能模块列表，通过树形结构勾选配置。 |
| 父子联动 | 勾选父级菜单时自动勾选子菜单，简化权限配置。 |
| 数据权限 | 控制角色可查看的数据范围，与组织架构关联。 |
| 分配用户 | 将角色权限赋予具体账号。一个角色可分配给多个用户，一个用户也可拥有多个角色。 |
| 取消授权 | 移除用户的角色权限，用户将失去该角色对应的菜单权限和数据权限。 |

## 常见问题

### Q：菜单权限和数据权限有什么区别？

A：菜单权限控制「能看到哪些功能页面」；数据权限控制「能看到哪些数据」。两者叠加生效，用户需要同时具备对应菜单权限和数据权限，才能查看相关数据。

### Q：一个用户可以同时拥有多个角色吗？

A：可以。一个用户可分配多个角色，权限按并集生效。建议避免角色之间权限边界冲突。

### Q：为什么用户登录后看不到某些菜单？

A：请检查用户是否已分配角色、角色是否启用，以及角色的菜单权限是否勾选了对应功能模块。

### Q：角色删除后，原分配用户会怎样？

A：删除角色前应先取消所有用户授权。角色删除后，用户将失去该角色对应的所有菜单权限和数据权限。
