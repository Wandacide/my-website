# 账号管理

本文介绍管理员如何维护已有账号，包括编辑账号信息、重置密码、分配角色、批量导入、导出数据和删除账号。

## 功能简介

账号管理用于保障系统账号的持续可用和权限准确。管理员可根据人员变动、岗位调整、部门调整和安全要求，对账号信息和权限进行维护。

常见使用场景包括：

- 员工部门或岗位发生变化，需要更新账号信息；
- 用户忘记密码，需要管理员重置；
- 用户职责变化，需要新增或移除角色；
- 需要批量导入账号或导出账号清单用于审计；
- 用户离职或不再使用系统，需要停用或删除账号。

## 操作前提

进行账号管理前，请确认：

1. 当前登录账号拥有「账号管理 - 创建账号」下的编辑、删除、重置密码、分配角色等权限；
2. 如需调整部门，需提前维护好组织架构；
3. 如需调整角色，需提前在「角色管理」中配置可用角色；
4. 删除账号前已确认该账号不再需要保留。

## 管理账号信息

### 1. 进入账号列表

在左侧导航栏中，选择「账号管理」-「创建账号」，进入用户列表。

[账号列表占位图]

> 图片占位：账号列表页面截图，建议展示搜索条件、用户列表和操作栏。

### 2. 编辑账号

在目标账号所在行，点击「编辑」，打开「编辑用户」弹窗。

可修改字段包括：

- 用户昵称；
- 归属部门；
- 手机号码；
- 邮箱；
- 用户性别；
- 状态；
- 岗位；
- 角色；
- 备注。

不可修改字段：

- 用户名称：即登录账号名，创建后不可修改。

[编辑用户占位图]

> 图片占位：编辑用户弹窗截图，建议展示可编辑字段和不可编辑的用户名称字段。

编辑完成后，点击「保存」。保存后配置即时生效。

## 分配角色

在账号列表中，点击目标账号操作栏的「分配角色」，进入分配角色页面。

操作方式：

1. 查看当前已分配角色，已分配角色通常为勾选状态；
2. 勾选需要新增的角色；
3. 取消勾选需要移除的角色；
4. 点击「提交」保存配置。

[分配角色占位图]

> 图片占位：账号分配角色页面截图，建议展示角色列表、多选框和「提交」按钮。

:::tip 权限生效规则
一个账号可以拥有多个角色。多个角色同时存在时，账号权限按并集生效，即用户可访问所有已分配角色授权的菜单与数据范围。
:::

## 重置密码

当用户忘记密码或存在密码泄露风险时，管理员可为账号重置密码。

操作方式：

1. 在账号列表找到目标账号；
2. 点击「重置密码」；
3. 输入新密码并保存；
4. 将新密码通过安全渠道告知用户。

## 批量导入账号

如需批量创建账号，可使用导入功能。

操作方式：

1. 点击「更多」-「下载模板」；
2. 按模板要求填写账号信息；
3. 点击「导入数据」并上传填写完成的模板；
4. 根据系统提示确认导入结果。

[批量导入占位图]

## 导出账号数据

如需进行权限盘点、审计留档或备份，可导出账号清单。

操作方式：

1. 点击「更多」-「导出数据」；
2. 系统下载用户信息清单；
3. 管理员可按需留存或进一步核对。

## 停用与删除账号

账号不再使用时，建议优先选择「停用」。

| 操作 | 影响 | 建议场景 |
| --- | --- | --- |
| 停用 | 账号无法登录，但历史数据和操作记录保留，可恢复。 | 员工暂时离岗、账号待确认、权限临时冻结。 |
| 删除 | 永久移除账号及关联信息，操作不可逆。 | 确认账号不再需要，且已完成数据留存与审计。 |

:::caution 操作建议
删除账号前请谨慎确认。若只是暂时禁止登录，建议使用「停用」而非删除。
:::

## 关键字段说明

| 字段 | 说明 |
| --- | --- |
| 用户编号 | 系统自动生成的唯一账号标识，不可编辑。 |
| 用户名称 | 登录账号名，用于系统认证，创建后不可修改。 |
| 用户昵称 | 账号显示名称，可随时修改。 |
| 归属部门 | 账号所属部门，关联数据权限范围。 |
| 手机号码 | 用于登录验证、短信通知或密码找回。 |
| 邮箱 | 用于邮件通知或密码找回。 |
| 状态 | 正常表示可登录；停用表示禁止登录。 |
| 岗位 | 预设岗位类型，用于业务区分。 |
| 角色 | 账号关联的权限角色，决定可访问功能。 |
| 创建时间 | 账号创建时间，用于审计追溯。 |

## 常见问题

### Q：忘记密码如何处理？

A：管理员可在账号列表点击「重置密码」设置新密码；用户也可通过登录页的忘记密码能力，使用绑定手机号或邮箱自助找回。

### Q：账号的部门可以更改吗？

A：可以。通过「编辑」修改归属部门后，相关数据权限将按照新部门重新生效。

### Q：为什么建议优先停用而不是删除？

A：停用可以保留历史数据和操作记录，便于审计和恢复；删除为不可逆操作，可能影响后续追溯。
